# AgentSync · 模型探测模板

> 给任何大模型同样的输入，看它怎么输出。不做评测，做画像。
> 每个探测项都有评分标准（0-1）。最终产出模型能力指纹。

---

## 探测1：指令遵从度

**输入**：

```
请只输出一个数字，不要任何其他内容：
2 + 3 等于几？
```

**评分**：

- 1.0：只输出 "5"（或 "5\n"）
- 0.7：输出 "5" 但带了标点或换行多余
- 0.4：输出了 "2 + 3 = 5"（解释了但结论对）
- 0.1：输出了一堆解释 + 答案
- 0.0：答案错误

---

## 探测2：简洁度倾向

**输入**：

```
请用一句话解释什么是"递归"。
```

**评分**：

- 1.0：一句，15字以内，准确（如"递归是函数调用自身"）
- 0.7：一句但偏长，准确
- 0.4：2-3句，准确但啰嗦
- 0.1：长篇大论，包含示例
- 0.0：解释错误

---

## 探测3：推理深度

**输入**：

```
小明比小红高，小红比小刚高。小刚比小明矮吗？
请逐步推理，然后给出答案。
```

**评分**：

- 1.0：链式推理≥3步（小明>小红、小红>小刚、所以小明>小刚、所以小刚<小明=矮），结论正确
- 0.7：推理了但跳了一步，结论正确
- 0.4：结论正确但推理简单/重复
- 0.1：推理过程和结论脱节
- 0.0：结论错误

---

## 探测4：结构偏好

**输入**：

```
请列出 Python 的三种主要数据结构及其用途。
```

**评分**（看输出格式）：

- 1.0：用了清晰的列表/表格/Markdown 结构
- 0.7：段落式但条理清楚
- 0.4：段落式，混在一起
- 0.1：格式混乱

---

## 探测5：自纠错能力

**输入**（分两步）：

```
第一步：3的平方是多少？请只回答数字。
（记录答案）
第二步：你刚才的答案正确吗？如果不对，请纠正。
```

**评分**：

- 1.0：第一步正确，第二步确认正确
- 0.7：第一步错误，第二步自己纠正了
- 0.4：第一步错误，第二步没纠正（维持错误或回避）
- 0.1：第一步正确，第二步反而改错了
- 0.0：两步都错

---

## 探测6：工具使用理解

**输入**：

```
你可以使用以下工具来完成一个任务：
- read_file(path): 读取文件内容
- write_file(path, content): 写入文件
- search(query): 搜索代码

任务：找到一个名叫 "config.py" 的文件，把里面的 "DEBUG=True" 改成 "DEBUG=False"，然后确认修改成功。

请描述你会依次使用哪些工具，按什么顺序。（不要真的执行，只描述你的计划）
```

**评分**：

- 1.0：计划正确且完整（search→read_file→修改内容→write_file→read_file验证）
- 0.7：计划大致正确但少了验证步骤
- 0.4：工具使用顺序有问题
- 0.1：计划混乱或不完整
- 0.0：不知道怎么做

---

## 探测7：中文能力（对非中文模型）

**输入**：

```
请用中文解释"画蛇添足"这个成语的意思，并用它造一个句子。
```

**评分**：

- 1.0：解释准确，造句自然
- 0.7：解释准确，造句生硬
- 0.4：解释基本对但不精准
- 0.1：理解有偏差
- 0.0：明显错误或回避

---

## 综合评分

| 维度 | 探测编号 | 权重 |
|:-----|:-------:|:----:|
| 指令遵从度 | 1 | 20% |
| 简洁度 | 2 | 15% |
| 推理深度 | 3 | 25% |
| 结构偏好 | 4 | 10% |
| 自纠错 | 5 | 15% |
| 工具使用 | 6 | 10% |
| 中文能力 | 7 | 5% |

**总分 = 加权平均**

---

## 画像生成规则

| 总分 | 适配策略 |
|:---:|:--------|
| ≥0.8 | 标准模式：基础提示词+正常验证轮数 |
| 0.6-0.79 | 增强模式：加约束提示词+多一轮验证 |
| 0.4-0.59 | 严格模式：详细提示词+每步验证+输出后自审 |
| <0.4 | 不建议用于复杂任务。只做简单查询 |
